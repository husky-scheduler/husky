# Security Policy

## Supported versions
Husky is currently in alpha. Security fixes may be released only for the latest prerelease and the most recent stable release, when available.

## Reporting a vulnerability
Please do not open a public GitHub issue for potential vulnerabilities.

Instead, report privately to the maintainers with:
- a description of the issue
- affected versions or commit hashes
- reproduction steps or proof of concept, if available
- impact assessment, if known

Until a dedicated security contact is published, use the repository owner or maintainer contact channel.

## Response goals
The project will try to:
- acknowledge receipt promptly
- validate the report
- coordinate a fix
- publish a remediation note when appropriate

Response times are best effort during alpha.

## Hardening guidance
Operational security guidance for Husky deployments is documented in [docs/security.md](docs/security.md).
